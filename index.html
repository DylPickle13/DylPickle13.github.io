EnGem

EnGem is a local assistant and automation framework that connects a streaming LLM (Gemini) to host-side tooling and exposes a Discord bot interface. It provides:

  • LLM orchestration and sub-agent execution plans
  • A Discord bot for text chat, scheduling, and execution-plan progress tracking
  • Persistent semantic memory backed by ChromaDB
  • Modular skills for browser automation, running code, notebooks, and media generation
  • Output capture and artifact relocation into generated_files/

This README shows how to get started, where important pieces live, and common development tasks.

Quick start

1. Create and activate a virtual environment, then install dependencies:

python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

2. Install Playwright browser binaries (required for browser automation):

python -m playwright install

3. Set required environment variables (examples):

  • DISCORD_BOT_TOKEN — Discord bot token (required to run the bot).
  • PAID_GEMINI_API_KEY — Gemini API key used by the LLM clients/tools.
  • DISCORD_ALLOWED_CHANNELS — Comma-separated allowed channel names (optional).
  • Optional: REPO_PATH, CRON_JOB_HOUR, CRON_JOB_MINUTE, HEARTBEAT_INTERVAL_SECONDS, MODEL — see config.py for defaults.

4. Run the Discord bot from the repository root:

python discord_bot.py

This starts the bot. The bot writes per-channel conversation history files to memory/channel_history/ and uses llm.py to orchestrate intent classification and sub-agent execution.

Tools and helpers

The repository contains several helper tools under tools/. Common helpers include:

  • tools/run_google_search.py — Google Search via Gemini.
  • tools/run_python.py — Safely run Python snippets and capture artifacts.
  • tools/run_notebook.py — Execute notebooks via Papermill.
  • tools/use_browser.py — High-level browser automation via ComputerUse.
  • tools/generate_image.py & generate_video.py — Media generation helpers.
  • tools/collect_generated_media.py — List recent outputs.

Data, storage, and runtime artifacts

  • Conversation histories: memory/channel_history/*.md (one file per Discord channel).
  • Vector DB: memory/vector_db/ (ChromaDB persistent client).
  • Generated outputs and artifacts: generated_files/.
  • Sub-agent execution orders: sub-agents/ (JSON files).
  • Agent instructions: agent_instructions/ (templates).

Troubleshooting

If Playwright actions fail, ensure browsers are installed (python -m playwright install).

If Gemini/Google APIs error frequently, api_backoff.call_with_exponential_backoff retries on transient errors; verify API keys.

If the bot does not receive messages, verify DISCORD_BOT_TOKEN and DISCORD_ALLOWED_CHANNELS.

Developer notes

  • Manager/Planner enforces a final `Reviewer` agent that prints <yes>.
  • Tool discovery: LLM sub-agents call helpers in tools/. Ensure valid function signatures.
  • Artifact relocation keeps the repo clean.
  • ChromaDB uses sentence-transformers. Back up memory/vector_db/.
